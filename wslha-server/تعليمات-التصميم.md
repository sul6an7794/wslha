# تعليمات لأداة التصميم (Claude Design) — تعديلات واجهة «وصّلها»

> **السياق:** يوجد **سيرفر جاهز ومُختبَر** (Node + Socket.io + MongoDB) فيه نظام حسابات وكريدت ولوحة تحكم.
> المطلوب منك تعديلات على **الواجهة فقط** (نفس التصميم والأسلوب الحالي: ثيم بنفسجي غامق، RTL عربي).
> كل نقاط الـ API موضّحة بالأسفل — وصّل الواجهة بها مباشرة.

---

## التغييرات المطلوبة

### 1) حفظ الجلسة (مهم — إصلاح)
حاليًا التوكن يُحفظ في الذاكرة فقط، فأي تحديث للصفحة يطلّع المستخدم.
- بعد نجاح **الدخول/التسجيل**: احفظ `token` و`user` في `localStorage` (مثلًا `wsl_token`, `wsl_user`).
- عند **فتح الصفحة**: استرجعهما، وأعد الاتصال بالسوكت تلقائيًا.
- عند **تسجيل الخروج**: احذفهما.

### 2) إزالة «المشرف الوهمي» (مهم — إصلاح)
حاليًا عند **فشل** تسجيل الدخول، الكود يزيّف حساب مشرف بدون توكن، فتظهر لوحة التحكم لكن ترجع خطأ 401.
- المطلوب: عند فشل الدخول/التسجيل، **أظهر رسالة الخطأ الفعلية** من السيرفر ولا تُسجّل دخول وهمي.

### 3) نظام الكريدت / المحفظة (بنفس أسلوب التطبيق)
لكل حساب **رصيد كريدت** (يبدأ بـ10، وكل لعبة تُنشأ تخصم 1).
- اعرض الرصيد بشكل أنيق **داخل التطبيق** (في الهيدر بجانب اسم المستخدم، وفي صفحة الحساب) — مو كطبقة منفصلة.
- مصدر الرصيد: `GET /api/auth/me` (يرجّع `user.credits`). حدّثه بعد كل لعبة.
- عند إنشاء غرفة، رد السوكت يحتوي `credits` (الرصيد المتبقي) — استخدمه للتحديث.
- إذا الرصيد لا يكفي، رد السوكت يكون `{ ok:false, error:'رصيدك لا يكفي لإنشاء لعبة جديدة' }` — اعرض الرسالة بشكل لطيف.

### 4) صفحة «حسابي» (جديدة)
عند الضغط على **اسم المستخدم** في الهيدر، افتح صفحة/نافذة فيها:
- **الرصيد المتاح** (من `/api/auth/me`).
- **تغيير الاسم:** حقل + زر → `PATCH /api/auth/profile { username }`. عند النجاح يرجع `token` جديد — احفظه (لأن الاسم مضمّن في التوكن).
- **تغيير كلمة المرور:** حقلان (الحالية + الجديدة) + زر → `PATCH /api/auth/profile { currentPassword, newPassword }`.
- اعرض رسائل النجاح/الخطأ القادمة من السيرفر (مثل «الاسم مستخدم من قبل»، «كلمة المرور الحالية غير صحيحة»).

### 5) صور اللاعبين الضبابية (إصلاح في اللعب الحقيقي)
حاليًا أثناء اللعب على السيرفر، كل لاعب يستلم **صورته فقط**، بينما تعرض الواجهة صور بقية اللاعبين من بيانات تجريبية محلية (فما تظهر صحيحة).
- المطلوب: اعرض صور بقية اللاعبين **مضببة** من بيانات السيرفر الفعلية.
- ⚠️ هذا يحتاج حدث سوكت إضافي من السيرفر (قائمة صور الجولة لكل اللاعبين). **سأضيفه أنا في السيرفر** — نسّقه معي: سيُرسل حدث `roundImages` يحتوي `[{ playerIndex, src }]`، وتعرض الواجهة الصور التي `playerIndex` لغير اللاعب الحالي بتأثير `blur`.

### 6) استقرار المؤقّت (إصلاح)
عند الاتصال بالسيرفر، يتعارض المؤقّت المحلي مع الوقت القادم من السيرفر فيقفز/يهتز.
- المطلوب: عند الاتصال بالسيرفر، **اعتمد على وقت السيرفر فقط** (حدث `state.elapsed`) ولا تُشغّل المؤقّت المحلي بالتوازي. المؤقّت المحلي للوضع غير المتصل فقط.

### 7) مقاسات الجوال (Responsive)
- تأكد من وجود `<meta name="viewport" content="width=device-width, initial-scale=1">`.
- راجع الأحجام الثابتة (مثل ارتفاع الصورة 260px، شبكة عدد اللاعبين، النوافذ) لتناسب شاشات الجوال الصغيرة دون أفقي scroll.

## عقد الـ API (السيرفر الجاهز)

**القاعدة:** كل المسارات على نفس أصل الموقع (same-origin). أرسل التوكن في `Authorization: Bearer <token>`.

### الحسابات
| الطريقة | المسار | الجسم | الرد |
|---|---|---|---|
| POST | `/api/auth/register` | `{username, password}` | `{token, user}` — أول مستخدم يصبح مشرفًا، يبدأ بـ10 كريدت |
| POST | `/api/auth/login` | `{username, password}` | `{token, user}` |
| GET | `/api/auth/me` | — (Bearer) | `{user}` |
| PATCH | `/api/auth/profile` | `{username?}` و/أو `{currentPassword, newPassword}` (Bearer) | `{user, token?}` |

شكل `user`: `{ id, username, isAdmin, credits }`.
أخطاء `/profile`: `409` الاسم مستخدم، `401` كلمة المرور الحالية غير صحيحة، `400` لا تغيير/كلمة قصيرة.

### لوحة التحكم (للمشرف فقط، Bearer)
| الطريقة | المسار | ملاحظات |
|---|---|---|
| GET | `/api/admin/rounds` | قائمة الجولات |
| POST | `/api/admin/rounds` | `{hint, answers, hintPlayerIndex}` (answers مفصولة بفواصل) |
| DELETE | `/api/admin/rounds/:id` | حذف جولة |
| POST | `/api/admin/rounds/:id/images` | رفع صور (multipart، الحقل `images`) — تُضغط وتُحفظ تلقائيًا |
| PATCH | `/api/admin/rounds/:id/images/:imageId` | `{playerIndex}` |
| GET | `/api/admin/stats` | `{activeRooms, totalPlayers, users, rounds}` |

### السوكت (Socket.io) — `io(origin, { auth: { token } })`
| الحدث (إرسال) | الجسم | الرد (callback) |
|---|---|---|
| `createRoom` | `{maxPlayers}` | `{ok, roomCode, teams, credits}` أو `{ok:false, error}` |
| `joinRoom` | `{roomCode}` | `{ok, roomCode, teams}` |
| `chooseTeam` | `{roomCode, teamIndex, teamName, name}` | `{ok, teams, teamIndex, isCaptain}` |
| `startGame` | — | `{ok}` |
| `submitAnswer` | `{answer}` | `{correct/locked, ...}` |

أحداث يبثّها السيرفر للواجهة: `lobby`, `gameStarted`, `state` (فيه `elapsed`, `score`, `roundIndex`), `yourImage` (`{src, hint}`), `correct`, `locked`, `finished`، **و(قريبًا) `roundImages`** للنقطة 5.

## ملاحظات الأسلوب
- حافظ على نفس الثيم: خلفية بنفسجية غامقة، تدرّجات `#818cf8 → #ec4899 / #9333ea`، اتجاه RTL، خط النظام.
- النصوص كلها عربية.

## بعد التصدير
عند تصدير الملف الجديد، أرسله لي وأنا:
1. أضعه في `server/public/index.html`.
2. أضبط `apiUrl` على **same-origin** (يتصل تلقائيًا بنطاق النشر بدل localhost).
3. أضيف حدث `roundImages` في السيرفر (للنقطة 5) وننسّق ربطه.
