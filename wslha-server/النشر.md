# نشر موقع «وصّلها» على الإنترنت (مجانًا) مع حفظ دائم للبيانات

تم تجهيز المشروع لينشر كـ **خدمة واحدة** على [Render](https://render.com) (خطة مجانية)،
مع تخزين دائم على **MongoDB Atlas** (مجاني) — فالجولات والحسابات والصور **لا تختفي** بعد إعادة التشغيل.

السيرفر (Node + Socket.io) يخدّم واجهة اللعبة بنفسه، فما تحتاج تنشر الواجهة بشكل منفصل ولا تعدّل أي عنوان.

> **ليش Render وليس Vercel؟** اللعبة تعتمد على اتصال لحظي دائم (Socket.io) ورفع صور،
> وهذا ما يشتغل على Vercel (serverless). Render (أو Railway) هو الخيار الصحيح.

---

## الخطوة 1: قاعدة بيانات دائمة مجانية (MongoDB Atlas)

1. سجّل في [mongodb.com/cloud/atlas/register](https://www.mongodb.com/cloud/atlas/register).
2. أنشئ **Cluster** مجاني (نوع **M0 Free**).
3. **Database Access** ← **Add New Database User**: اختر اسم مستخدم وكلمة مرور (احفظهم).
4. **Network Access** ← **Add IP Address** ← **Allow Access from Anywhere** (`0.0.0.0/0`).
5. **Clusters** ← **Connect** ← **Drivers** ← انسخ رابط الاتصال، شكله:
   ```
   mongodb+srv://USER:PASSWORD@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```
   - استبدل `USER` و`PASSWORD` ببيانات الخطوة 3.
   - أضف اسم قاعدة بيانات قبل علامة `?` هكذا: `.../teamquest?retryWrites=...`

احتفظ بهذا الرابط — ستضعه في Render كمتغيّر `MONGODB_URI`.

> بدون `MONGODB_URI` يشتغل الموقع لكن بتخزين مؤقت (يختفي عند إعادة التشغيل). أضِف الرابط للحفظ الدائم.

## الخطوة 2: ارفع المشروع على GitHub
تم بالفعل الرفع على: <https://github.com/sul6an7794/wslha>
(لو عدّلت شيئًا لاحقًا: `git add -A && git commit -m "..." && git push`).

## الخطوة 3: أنشئ الخدمة على Render
1. ادخل [dashboard.render.com](https://dashboard.render.com) → **New** → **Blueprint**.
2. اربط GitHub واختر مستودع **`wslha`** → سيقرأ `render.yaml` تلقائيًا.
3. قبل **Apply** (أو بعده من تبويب **Environment**)، أضف متغيّر البيئة:
   - **`MONGODB_URI`** = رابط الاتصال من الخطوة 1.
   - (`JWT_SECRET` يولّده Render تلقائيًا.)
4. اضغط **Apply** وانتظر اكتمال البناء (~2-3 دقائق).

> بدون Blueprint: **New → Web Service**، اختر المستودع، واضبط يدويًا:
> **Root Directory:** `server` · **Build:** `npm install` · **Start:** `npm start` · **Health Check Path:** `/health`
> ثم أضف متغيّري البيئة `MONGODB_URI` و`JWT_SECRET`.

## الخطوة 4: جاهز
يعطيك Render رابطًا مثل `https://team-quest-xxxx.onrender.com`.
افتحه — تشتغل الواجهة وتتصل بالسيرفر تلقائيًا. **أول حساب يسجّل يصبح المشرف**؛ سجّل حسابك أولًا ثم افتح لوحة التحكم لإضافة الجولات والصور. كل شيء يُحفظ دائمًا في MongoDB. ✅

تأكيد التخزين: في سجلّات Render (Logs) يجب أن تظهر `التخزين: MongoDB (دائم) ✅`.

## ملاحظات
- **الصور:** تُخزَّن داخل MongoDB (الحد 12MB لكل صورة) وتُخدَّم عبر `/img/...` — تبقى بعد إعادة التشغيل.
- **النوم:** الخطة المجانية في Render تنام بعد ١٥ دقيقة خمول؛ أول طلب بعدها يأخذ ~٥٠ ثانية (cold start). لا يؤثر على حفظ البيانات.
- **حصة Atlas المجانية:** 512MB تكفي لآلاف الجولات والصور المتوسطة.

## التشغيل المحلي (للتجربة)
```
cd server
cp .env.example .env      # وعدّل القيم؛ اترك MONGODB_URI فاضيًا لتخزين محلي بملف، أو ضع رابط Atlas
npm install
npm start
```
ثم افتح <http://localhost:3001>.

## تحديث الواجهة لاحقًا
الواجهة المنشورة هي `server/public/index.html`. لو صدّرت تصميمًا جديدًا من أداة التصميم،
انسخه فوق هذا الملف (مع إبقاء `default` لـ `apiUrl` فارغًا) ثم `git push` لإعادة النشر تلقائيًا.
