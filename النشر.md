# نشر موقع «وصّلها» على الإنترنت (مجانًا)

تم تجهيز المشروع لينشر كـ **خدمة واحدة** على [Render](https://render.com) (الخطة المجانية):
السيرفر (Node + Socket.io) يخدّم واجهة اللعبة بنفسه، فما تحتاج تنشر الواجهة بشكل منفصل ولا تعدّل أي عنوان.

> **ليش Render وليس Vercel؟** اللعبة تعتمد على اتصال لحظي دائم (Socket.io) ورفع صور للسيرفر،
> وهذا ما يشتغل على Vercel (serverless). Render (أو Railway) هو الخيار الصحيح.

---

## الخطوات

### 1) ارفع المشروع على GitHub
أنشئ مستودعًا جديدًا على GitHub وارفع محتوى مجلد `teamquest2` كاملًا (فيه `render.yaml` و`server/`).

### 2) أنشئ الخدمة على Render
1. ادخل [dashboard.render.com](https://dashboard.render.com) → **New** → **Blueprint**.
2. اربط حساب GitHub واختر المستودع. سيقرأ Render ملف `render.yaml` تلقائيًا.
3. اضغط **Apply**. سينشئ خدمة باسم `team-quest` ويولّد `JWT_SECRET` تلقائيًا.

> بدون Blueprint: **New → Web Service**، اختر المستودع، واضبط يدويًا:
> - **Root Directory:** `server`
> - **Build Command:** `npm install`
> - **Start Command:** `npm start`
> - **Health Check Path:** `/health`
> - أضف متغيّر بيئة `JWT_SECRET` بقيمة نصية طويلة عشوائية.

### 3) جاهز
بعد انتهاء النشر يعطيك Render رابطًا مثل `https://team-quest-xxxx.onrender.com`.
افتحه — تشتغل الواجهة فورًا وتتصل بالسيرفر تلقائيًا (نفس النطاق).

### 4) حساب المشرف
**أول حساب يسجّل في الموقع يصبح المشرف تلقائيًا.** سجّل حسابك أول واحد، وبعدها افتح لوحة التحكم لإضافة الجولات والصور.

## ⚠️ مهم: البيانات على الخطة المجانية

الخطة المجانية في Render **لا تحفظ الملفات بشكل دائم** (نظام ملفات مؤقت). معناه:

- عند كل إعادة تشغيل أو إعادة نشر: يُحذف `data.json` ومجلد `uploads/`.
- تُعاد إضافة **4 جولات افتراضية** تلقائيًا (عبر `seed.js`)، لكن:
  - الجولات والصور اللي تضيفها من لوحة التحكم **تختفي**.
  - الحسابات المسجّلة **تختفي** (وأول من يسجّل بعدها يصير مشرفًا من جديد).
- الخدمة المجانية **تنام بعد ١٥ دقيقة خمول**، وأول طلب بعدها يستغرق ~٥٠ ثانية (cold start).

**حلول الحفظ الدائم (اختياري):**
- ترقية خطة Render وإضافة **Persistent Disk** يُركَّب على مسار `server/` (يحفظ `data.json` و`uploads/`).
- أو تحويل التخزين من ملف JSON إلى قاعدة بيانات سحابية مجانية (مثل MongoDB Atlas) ورفع الصور على خدمة تخزين (مثل Cloudinary). أخبرني لو تبي أنفّذ هذا.

للعب العادي (جلسة واحدة في نفس الوقت) الوضع الحالي كافٍ.

## التشغيل المحلي (للتجربة قبل النشر)
```
cd server
npm install
npm start
```
ثم افتح <http://localhost:3001> في المتصفح.

## تحديث الواجهة لاحقًا
الواجهة المنشورة هي `server/public/index.html`. لو عدّلت التصميم وصدّرت ملف HTML جديدًا،
انسخه فوق `server/public/index.html` (مع إبقاء `default` لـ `apiUrl` فارغًا) ثم أعد النشر.
