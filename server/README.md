# سيرفر Team Quest

باك إند حقيقي للعبة: تسجيل دخول، غرف لحظية عبر Socket.io، ولوحة تحكم لإضافة الجولات والصور. البيانات تُحفظ في ملف `data.json` محلي (تخزين JS خالص، بدون أي اعتمادية تحتاج تجميع/compile — يعمل بدون Python أو Visual Studio Build Tools).

## التشغيل المحلي

**الطريقة الأسهل:** دبل كلك على `start.bat` داخل هذا المجلد — يسوي كل شي تلقائيًا (تثبيت + تشغيل).

**أو يدويًا:**
1. ثبّت [Node.js](https://nodejs.org) (نسخة LTS) إذا ما كان مثبت عندك.
2. افتح Terminal/PowerShell داخل مجلد `server` وشغّل:
   ```
   npm install
   npm start
   ```
3. لو ظهرت رسالة `Team Quest server running on http://localhost:3001` فالسيرفر شغّال.

**لو واجهت خطأ npm install قبل كذا (متعلق بـ better-sqlite3 أو Python أو node-gyp):** هذا محلول الآن — التخزين تغيّر لـ JSON خالص. لو عندك مجلد `node_modules` أو ملف `package-lock.json` من محاولة سابقة، احذفهم قبل إعادة التشغيل (أو فقط استخدم `start.bat` فهو يحذفهم تلقائيًا قبل كل تثبيت).

تصميم الواجهة (الملف `Team Quest - نسخة مستقلة.html`) مضبوط مسبقًا على `http://localhost:3001` كعنوان للسيرفر — يعني ما تحتاج تغيّر أي شي بالواجهة للتجربة المحلية. فقط:

1. شغّل السيرفر كما بالأعلى واتركه يعمل.
2. افتح ملف الواجهة في المتصفح (بنفس طريقة فتح الملف اللي استخدمناها قبل: Ctrl+O داخل المتصفح).
3. سجّل حساب جديد من شاشة "تسجيل الدخول" — أول حساب يسجّل بهذا السيرفر يصبح **مشرف (Admin)** تلقائيًا ويشوف رابط "لوحة التحكم".
4. جرّب إنشاء غرفة أو الانضمام لغرفة من جهاز/متصفح ثاني (لازم يكون نفس الشبكة أو نفس الجهاز) للتأكد من أن اللعب الجماعي يشتغل عبر السيرفر الحقيقي.

السيرفر جاهز فيه 4 جولات تجريبية بصور افتراضية حتى تقدر تجرب اللعبة فورًا بدون إضافة جولات يدويًا.

**الإجابة إجبارية، التلميح اختياري ولاعب واحد فقط:** عند إنشاء جولة، الإجابة (إجابة واحدة على الأقل) مطلوبة دائمًا — القائد يكتبها بعد ما يتفق الفريق شفهيًا. التلميح اختياري، ولو حطيته فهو يروح **فقط للاعب اللي تحدد رقمه** بخانة "التلميح لِلاعب رقم" (مثلاً: لاعب رقم 2 يشوف نص التلميح، وباقي اللاعبين ما يشوفون شي إطلاقًا). هذا يسمح بجولات "اختلاف الصور" حيث لاعب واحد عنده دليل إضافي يساعد فريقه بدون ما يصرّح به مباشرة.

**تحديد أي لاعب يستلم أي صورة:** عند رفع صور لجولة، أول صورة تروح تلقائيًا للاعب رقم 1، الثانية للاعب 2، وهكذا (حسب ترتيب الرفع). بلوحة التحكم تقدر تغيّر هذا يدويًا — تحت كل صورة فيه خانة رقم صغيرة، غيّرها للاعب الذي تريد وتُحفظ تلقائيًا.

> **ملاحظة عن الواجهة:** ميزتي "تلميح لِلاعب واحد فقط" و"تحديد لاعب كل صورة" مفعّلتان بالكامل بالباك إند (هذا المجلد). أما خانات التحكم بهما داخل لوحة التحكم بالواجهة — فحاليًا غير موجودة بملفي المصدر `Team Quest.dc.html` و `Team Quest - Standalone.dc.html` (رجعت الملفات لنسخة أقدم بعد إعادة تصدير سابقة من Claude design وفقدت هذا الجزء من التعديل). الإجابة الإجبارية والتلميح يعملان من جهة السيرفر بأي حال، لكن لو احتجت خانة "التلميح لِلاعب رقم" بنفس واجهة لوحة التحكم تحتاج إعادة هذا التعديل بالواجهة مرة ثانية.

> **ملاحظة عن حفظ تعديلات الواجهة:** Claude design يحتفظ بحالة المشروع عنده، ولاحظنا إن إعادة تصدير نسخة جديدة من هناك قد ترجّع ملفاتك المحلية (`*.dc.html`) لنسخة أقدم وتفقد تعديلات سابقة لم تُحفظ هناك. الأفضل دائمًا: لما تسوي أي تعديل يدوي على ملف `.dc.html` محلي، الصقه فورًا داخل مشروع Claude design (نفس المحادثة) عشان يصير هو نفسه مصدر الحالة، بدل الاعتماد على نسخك المحلية وحدها.

## بنية المشروع

```
server/
  src/
    server.js      نقطة البداية (Express + Socket.io)
    db.js           تخزين البيانات في data.json (مستخدمون، جولات، صور)
    seed.js         بيانات تجريبية تُضاف تلقائيًا أول مرة فقط
    auth.js         تشفير كلمات المرور وتوقيع/فحص التوكن (JWT)
    rooms.js        منطق الغرف واللعبة اللحظية (في الذاكرة)
    socket.js       ربط أحداث Socket.io بمنطق الغرف
    routes/auth.js   /api/auth/register و /api/auth/login
    routes/admin.js  /api/admin/rounds, /api/admin/stats, رفع الصور
  uploads/          الصور المرفوعة من لوحة التحكم (تُنشأ تلقائيًا)
  data.json         قاعدة البيانات (تُنشأ تلقائيًا أول تشغيل)
```

## نقاط الـ API

- `POST /api/auth/register` `{ username, password }`
- `POST /api/auth/login` `{ username, password }`
- `GET /api/admin/rounds` (يتطلب توكن مشرف)
- `POST /api/admin/rounds` `{ hint, answers, hintPlayerIndex }` — `answers` مطلوب (نص مفصول بفواصل)، `hint` و `hintPlayerIndex` اختياريان (التلميح يظهر فقط للاعب بهذا الرقم، الافتراضي 1)
- `DELETE /api/admin/rounds/:id`
- `POST /api/admin/rounds/:id/images` (multipart, حقل `images`) — أول صورة تُرفع تروح تلقائيًا للاعب 1، الثانية للاعب 2، وهكذا
- `PATCH /api/admin/rounds/:id/images/:imageId` `{ playerIndex }` — لتحديد/تغيير أي لاعب يستلم صورة معيّنة يدويًا
- `GET /api/admin/stats`

## نظام الفرق المتعددة داخل نفس الغرفة

كود الغرفة الواحد ينقسم تلقائيًا لعدة فرق بحجم **3 لاعبين بالضبط** لكل فريق (`عدد الفرق = الحد الأقصى للاعبين ÷ 3`). كل فريق مستقل تمامًا عن باقي فرق نفس الغرفة:

- اللعبة لفريق معيّن **لا تبدأ إلا بعد اكتمال 3 لاعبين بهذا الفريق تحديدًا** — ما يشترط اكتمال أي فريق آخر بنفس الغرفة.
- أول لاعب يدخل فريقًا فاضيًا يصبح **قائده** ويسمّيه؛ من يدخل بعده ينضم له تلقائيًا كعضو (بدون تسمية مرة ثانية) إلى أن يكتمل.
- التوقيت/النقاط/رقم الجولة/التلميح/توزيع الصور — كل هذا محسوب لكل فريق لحاله، عبر قناة Socket.io خاصة بصيغة `roomCode:teamIndex`.
- لما فريق ينتهي من كل الجولات، باقي فرق نفس الغرفة يستمرون بلعبهم لحالهم بدون أي تأثير.

## أحداث Socket.io

- إرسال: `createRoom` `{ maxPlayers }`, `joinRoom` `{ roomCode }` (تحقق فقط من وجود الغرفة وإرجاع حالة فرقها), `chooseTeam` `{ roomCode, teamIndex, teamName?, name }` (اختيار/الانضمام لفريق), `startGame`, `submitAnswer` `{ answer }`
- استقبال: `lobby` `{ roomCode, teams }` (حالة كل فرق الغرفة — يُبث عند أي تغيير), `gameStarted`, `state`, `yourImage`, `correct`, `locked`, `finished` (آخر 5 أحداث خاصة بفريقك فقط)

## النشر أونلاين (خطوة لاحقة بعد التجربة المحلية)

هذا السيرفر يحتاج استضافة تدعم Node.js مستمر (وليس استضافة استاتيك مثل Netlify Drop)، لأن Socket.io يحتاج اتصال مستمر. خيارات مناسبة: Render أو Railway أو Fly.io (لها مستويات مجانية/رخيصة).

بعد ما يصير عندك رابط عام للسيرفر (مثلاً `https://your-app.onrender.com`)، رجع لتصميم Claude design وغيّر خاصية `apiUrl` لهذا الرابط، ثم صدّر نسخة "مستقلة" جديدة وأعد رفعها بنفس طريقة Netlify Drop. بدون هذا التحديث، النسخة المنشورة على الإنترنت تبقى تحاول الاتصال بـ `localhost` (يعني تعمل وضع محلي فقط لكل زائر، بدون لعب جماعي حقيقي).
